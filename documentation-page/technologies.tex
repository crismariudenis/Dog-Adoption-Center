The Dog Adoption Management System is built using a modern, cloud-native stack designed for high availability and scalability. The selection of these technologies was driven by the need to handle complex business logic while maintaining a professional user experience.

\section{Microservices and Clean Architecture}
The project adopts a \textbf{Microservices Architecture}, which allows each functional area to be developed and scaled independently \cite{microservices_fowler}. Within each service, we implement \textbf{Clean Architecture} \cite{clean_architecture}. This ensures that the core business logic is decoupled from external technical concerns like databases or web frameworks.

\section{Backend: .NET 10}
The backend is built using \textbf{.NET 10} \cite{microsoft_net10}. .NET was chosen for its high performance, robust NuGet ecosystem, and first-class support for Azure integrations. We use \textbf{Entity Framework Core} for ORM and \textbf{MediatR} to implement the CQRS pattern.

\section{Frontend: React}
The UI is a Single Page Application (SPA) built with \textbf{React} and \textbf{Vite}. We use \textbf{Tailwind CSS} for a utility-first approach to styling, ensuring the platform is responsive and consistent.

\section{Polyglot Persistence}
We utilize a polyglot persistence strategy:
\begin{itemize}
    \item \textbf{PostgreSQL:} For structured, relational data like user profiles and reviews.
    \item \textbf{Azure CosmosDB:} For highly scalable, document-based adoption applications \cite{cosmosdb_whitepaper}.
\end{itemize}

\section{Orchestration: Docker and AKS}
All components are containerized using \textbf{Docker}. The system is orchestrated by \textbf{Azure Kubernetes Service (AKS)}, which provides self-healing, auto-scaling, and a managed control plane for the microservices.

\section{DevOps and CI/CD}
We use \textbf{GitHub Actions} for automated CI/CD. The pipeline is secured using \textbf{OIDC (OpenID Connect)} for password-less authentication between GitHub and Azure. On every push to the main branch, all 8 container images are built, pushed to the \textbf{Azure Container Registry}, and deployed to the cluster.

\section{Future Technology Stack: AI and Mobile}
To realize our vision for a breed-scanning mobile feature, we plan to incorporate:
\begin{itemize}
    \item \textbf{Computer Vision (TensorFlow/PyTorch):} For implementing Convolutional Neural Networks (CNNs) that can classify dog breeds from mobile camera streams.
    \item \textbf{Mobile Development (React Native / Flutter):} To extend our web-based ecosystem into a cross-platform mobile application.
    \item \textbf{Identity and Document Verification (Azure AI Vision):} To automate the "convoluted" document verification process, ensuring that submitted IDs and housing permits are authentic and valid.
\end{itemize}

\section{Monitoring and Observability}
\textbf{Azure Application Insights} is integrated into the core services to provide real-time telemetry. This allows the team to monitor "Success Rates" of applications and identify performance bottlenecks in the distributed system.
